# la-rank — Rank & the Four Subspaces

## Current summary (what it teaches + what the lab does)
The lesson (`/Users/wedd/dev/ai-math-worktrees/mystifying-clarke-7ceb80/lib/curriculum/la-depth.js`, lesson id `la-rank`, order 6.2) introduces rank as the dimension of the column space (the set of all outputs `Ax`), walks through the rank-2 / rank-1 / rank-0 cases of a 2×2 matrix, defines the null space `{x : Ax = 0}`, states rank–nullity (`rank + dim(null) = n`), names all four fundamental subspaces (row space + null space partition the input ℝⁿ; column space + left null space partition the output ℝᵐ), and ties rank to determinant/invertibility ("rank 2 ⇔ det ≠ 0 ⇔ invertible"). The `ml` note frames rank as the "budget" of a linear layer and previews LoRA + autoencoder bottlenecks. The two `deeper` cards give a "shadow/projector" intuition for rank-1 and a clean statement of the four-subspace dimensions (r, n−r, r, m−r) attributed to Strang's Fundamental Theorem.

The lab (`INTERACTIVES.larank`) is a 2×2 transformation playground: four sliders set entries a,b,c,d; the unit circle's image (cyan dots) flattens from ellipse → segment → point as rank drops; columns are drawn as gold arrows; a draggable purple probe `x` shows its image `Ax`. Rank is computed numerically from the two singular values (closed-form 2×2) with a fixed threshold of 0.045. When rank = 1, the lab draws the column space (gold dashed line, along the surviving column) and the null space (red dashed line, perpendicular to the row direction). Three gated missions: (1) make a nonzero rank-1 matrix; (2) drag the probe into the null space so ‖x‖≥1 and ‖Ax‖≤0.1; (3) re-inflate to rank 2 with |det|≥2. Readout shows A, det, rank, both σ's, and ‖Ax‖. Presets: Identity, Shear, Squash.

## Strengths
- The "apply A to every vector and watch where outputs land" framing for column space is exactly the 3Blue1Brown geometric intuition and is correct.
- Rank–nullity is stated with the right variable (`n` = columns), and the missions force the learner to *physically* drag a vector into the null space and watch `Ax` die — a genuinely good kinesthetic for "the crushed directions."
- Tying σ's in the readout to the next SVD lesson is good scaffolding; numerical rank from singular-value thresholding is the professionally correct way to define rank in floating point.
- The shadow/projector `deeper` card is a strong, memorable intuition for information loss and non-invertibility.
- The embedding-matrix quiz question (rank ≤ min(m,n) = 512) is a genuinely ML-relevant application, not a toy.

## Inaccuracies / fidelity issues (each: the issue -> the correct statement -> source URL)
1. **The lab's null-space line is mislabeled/derived in a way that hides the actual theorem.** The null space line is drawn as perpendicular to the *row* `[a,b]` (`nl=[-row[1],row[0]]`). This is numerically *correct* for a rank-1 matrix (the null space is exactly the orthogonal complement of the row space), but the lesson never tells the learner *why* the null space is perpendicular to the rows — which is the single most important structural fact of the whole topic. The Fundamental Theorem says **row space ⊥ null space in ℝⁿ** and **column space ⊥ left null space in ℝᵐ**. Right now the learner sees a red line appear perpendicular to something and is given no reason. -> State explicitly: "Ax = 0 means x is orthogonal to every row of A, so the null space is exactly the orthogonal complement of the row space." -> https://web.mit.edu/18.06/www/Essays/newpaper_ver3.pdf and https://en.wikipedia.org/wiki/Fundamental_theorem_of_linear_algebra

2. **"row space (what the input actually gets measured along)" is loose and slightly misleading.** The row space is the set of linear combinations of the rows; geometrically the rows are the "measurement directions." But the phrase invites the misconception that the row space lives in the output. The row space lives in ℝⁿ (input side), and it is the orthogonal complement of the null space. -> "The row space lives in input space ℝⁿ alongside the null space: it's the part of x the matrix actually responds to, and it's perpendicular to the null space." -> https://en.wikipedia.org/wiki/Row_and_column_spaces

3. **"rank 2 ⇔ det ≠ 0 ⇔ invertible" is stated as a general fact but only holds for square matrices.** The lesson is mostly framed around 2×2, but the `deeper` card and the embedding quiz are explicitly about m×n. det and invertibility are square-only notions; rank is defined for any shape. A learner could carry "det ≠ 0 ⇔ full rank" into a rectangular setting where det doesn't exist. -> Add a one-clause guard: "(for a square matrix) full rank ⇔ det ≠ 0 ⇔ invertible; for non-square matrices there is no determinant and no two-sided inverse." -> https://en.wikipedia.org/wiki/Rank_(linear_algebra)

4. **Rank–nullity is presented only as `rank + dim(null) = n`, with no mention that column rank = row rank.** The fact that "the number of independent columns equals the number of independent rows" is *why* there is a single number called rank — it is the deepest and least obvious fact in the lesson, and it underpins the symmetric "r, n−r, r, m−r" table the `deeper` card presents. The lesson asserts the table but never flags that the two r's being equal is a theorem, not a definition. -> Add: "Remarkably, the column rank always equals the row rank — that's why one number ('the rank') describes both sides." -> https://en.wikipedia.org/wiki/Rank_(linear_algebra)

5. **Numerical-rank threshold is a fixed absolute 0.045, not relative.** This is fine for a sandbox where entries are bounded to [−2,2], but it is the opposite of professional practice and a master's learner will later meet `numpy.linalg.matrix_rank`, which uses a *relative* tolerance `σ_max · max(m,n) · ε`. Worth a one-line `deeper` note so the learner isn't surprised that "rank" is a tolerance-dependent decision in practice. -> "In real code, rank is decided by a tolerance: a singular value counts as nonzero only if it exceeds ~`σ_max · max(m,n) · ε`. Rank is a *numerical* judgment, not a yes/no fact, once floats are involved." -> https://numpy.org/doc/stable/reference/generated/numpy.linalg.matrix_rank.html

## Conceptual gaps (what a serious learner still needs)
- **Orthogonality of the four subspaces** — the defining content of Strang's Fundamental Theorem (Part 2) is entirely absent from the prose. The lab even *draws* the orthogonality (null ⊥ column-direction-of-rows) but never names it. This is the #1 gap.
- **The left null space is named but never shown or motivated.** A learner finishes with no concrete picture of it. For a 2×2 rank-1 matrix it's a real, visible red line in the *output* plane (directions no output ever reaches) — the lab could show it directly.
- **rank(AB) ≤ min(rank A, rank B)** and **rank(AᵀA) = rank(A)** — both are heavily used in ML (e.g., why a low-rank bottleneck caps the rank of everything downstream; why Gram matrices and normal equations preserve rank). Neither appears.
- **The "single gap in singular values" picture of numerical rank** — real weight matrices/data don't have hard zeros; they have a *spectrum that decays*, and "rank" is where you choose to cut. This is the bridge to SVD/PCA/LoRA and is more honest than the rank-0/1/2 trichotomy.
- **Connection rank → solvability of Ax = b** (consistent iff b ∈ column space; unique iff null space = {0}) — the lesson has all the pieces but never assembles the "existence + uniqueness" payoff that motivates the whole subject.
- **Effective/numerical rank of real ML matrices** — a sentence with a real datum (e.g., trained transformer weight matrices being well-approximated at a fraction of full rank) would make the `ml` note land harder and set up LoRA.

## Lab ideas (concrete: name the interactive, what the learner manipulates, what it reveals)
1. **Show the left null space, and prove orthogonality on screen (smallest change, biggest payoff).** In `INTERACTIVES.larank`, when rank = 1, additionally draw the *left null space* as a second red line in output space (perpendicular to the column direction `[a,c]`) labeled "no output ever lands here." Then add a faint right-angle marker between the gold column-space line and the red null-space line so the learner *sees* the orthogonality the theorem promises. This turns the existing red line from "mystery perpendicular" into "the Fundamental Theorem, on screen."
2. **"Decompose x" probe.** When the learner drags `x`, render its two components: the projection onto the row space (drawn in gold, "the part that survives") and the projection onto the null space (drawn in red, "the part that dies"), with a live readout `x = x_row + x_null`. Reveal: every input splits into a measured part and an ignored part, and only `x_row` contributes to `Ax`. Mission: "Make `Ax` unchanged while moving `x` — slide it along the null line." This makes rank–nullity tangible as a decomposition, not just an equation.
3. **Singular-value spectrum slider (bridge to SVD/low-rank).** Add a tiny two-bar chart of σ₁, σ₂ next to the readout (you already compute both). Add a "near-rank-1" preset like `[[2, 4], [1, 2.05]]` whose σ₂ is small-but-nonzero, and a slider for the rank tolerance. Reveal: rank flips from 2 to 1 as you raise the tolerance past σ₂ — i.e., "rank is a decision about how small is zero." Directly seeds the next lesson and the numerical-rank gap above.
4. **Solve `Ax = b` overlay.** Let the learner place a target `b` and shade the column-space line; show "solvable / not solvable" and, when solvable and rank-deficient, draw the whole *line* of solutions `x* + null`. Reveal: rank deficiency = infinitely many solutions (or none), connecting rank to the existence/uniqueness story.

## Content improvements (specific learn/ml/deeper text upgrades)

Two perpendicular pieces on each side — that's Strang's Fundamental Theorem." This is the lesson's missing spine.
- **`learn`:** Guard the determinant claim: change "rank 2 ⇔ det ≠ 0 ⇔ invertible" to "for a square matrix: full rank ⇔ det ≠ 0 ⇔ invertible (non-square matrices have no determinant)."
- **`learn`:** Add one clause flagging column rank = row rank as the surprising theorem that justifies the word "rank."
- **`ml`:** Strengthen with a concrete, sourced datum and the `rank(AB) ≤ min` lever: "Empirically the *weight update* during fine-tuning is well-approximated at very low rank — that's LoRA's bet (ΔW = BA). And because `rank(AB) ≤ min(rank A, rank B)`, a single low-rank bottleneck caps the rank of everything that flows through it — that's why autoencoder pinch-points force compression."
- **`deeper`:** Add a third card "Rank is a tolerance, not a fact" covering numerical rank (`σ_max · max(m,n) · ε`) and the decaying-spectrum picture of real matrices — honest framing that sets up SVD/PCA/LoRA.

## Quiz improvements (specific misconceptions to target; keep questions self-contained — never require recalling lab-graph data)
- **Existing Q1–Q4 are solid and self-contained — keep them.** The WRONG_WHY entries are unusually good (specific, concrete). No factual problems.

This targets the lesson's biggest gap and is fully self-contained.
- **Add: rank is shape-bounded on both sides + det is square-only.** "A 3×5 matrix is told to be 'invertible because its determinant is nonzero.' What's wrong?" Targets the det-only-for-square misconception flagged above.
- **Add: rank of a product.** "If A is rank 2 and B is rank 5, the rank of AB is at most ___." Options: 2 (correct) / 5 / 7 / 10. WRONG_WHY for 5: "rank(AB) ≤ min(rank A, rank B); the *smaller* operand is the bottleneck — squeezing through a rank-2 map can't restore rank."
