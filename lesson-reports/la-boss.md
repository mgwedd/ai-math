# la-boss — BOSS: Build a Neural Layer

## Current summary (what it teaches + what the lab does)
This is the capstone ("BOSS") of World 1 (Linear Algebra). The stated thesis: a neural layer computes `new_representation = W·x (+ bias, + nonlinearity)`, and the deep idea is that **layers re-represent data until a trivial rule separates the classes**. The `learn` text frames the learner as "being the weight matrix": tune W's four numbers so a vertical line `x = 0` separates two classes. The `ml` note generalizes this to the pixel→edge→shape→"cat vs dog" chain and ties it back to determinant/eigen ideas (when maps preserve vs. destroy information).

**Lab (`INTERACTIVES['boss-la']`):** Two fixed, jittered point clusters — class A (purple) centered near (1, 1.8), class B (orange) near (1.8, −0.8). The learner drags four sliders for the entries of a 2×2 matrix `M = [[a,b],[c,d]]` (labeled W₁₁…W₂₂, each in [−2, 2]). Each point `p` is mapped by `T(p) = (a·x + b·y, c·x + d·y)` — a **purely linear** map (no bias, no nonlinearity). The decision rule is the sign of the transformed x-coordinate. Readout shows W and `det = ad − bc`.

Three missions:
1. Separate: all purple at transformed `x < −0.2`, all orange at `x > 0.2` (+35 XP).
2. Do it while invertible: `|det| > 0.3` (+20 XP).
3. "Sabotage check": set `det ≈ 0` (|det| < 0.06) and watch both classes collapse onto one line (+15 XP).

The quiz has 3 questions (transform-to-separate, training adjusts W's entries, det=0 collapse risk), each with WRONG_WHY explanations in the `la-boss` block.

## Strengths
- **The det=0 sabotage mission is genuinely excellent and is directly endorsed by the canonical source.** Olah's "Neural Networks, Manifolds, and Topology" makes exactly this point: when the weight matrix is non-invertible, "the dataset gets collapsed on some axis," irreversibly mixing classes. Letting the learner *cause* that collapse with sliders is high-fidelity, hands-on intuition for why invertibility/`det≠0` matters.
- **"You are the optimizer" framing is pedagogically strong** — it makes concrete what gradient descent will later automate, and the quiz reinforces that *the weights are the learnable thing* (a real beginner confusion: people think training changes the data or the labels).
- **Honest forward-pointer to World 2** ("gradient descent finds these numbers automatically") sets up the next world well.
- **Ties back to prior World-1 concepts** (determinant, information loss) so the boss actually feels like a synthesis.

## Inaccuracies / fidelity issues (each: the issue -> the correct statement -> source URL)

1. **The lab is the lesson's biggest fidelity gap: it teaches a *linear classifier*, not *representation learning*, yet the prose claims it shows "what learning IS."** The two clusters in the lab are **already linearly separable** in raw space (purple is up-and-left, orange is down-and-right — a single oblique line splits them). A linear map W can only **rotate/scale/shear** that already-separable data so the existing separating line aligns with `x=0`. It cannot make a *non*-separable dataset separable. → The whole point of a hidden layer is that **composition of linear maps is still linear and adds no expressive power**; you need a **nonlinearity** to untangle classes that no straight line can separate. "Adding linear layers does not change the capacity or expressivity of a network… stacking multiple linear transformations doesn't add expressive power beyond a single linear transformation." The lab demonstrates the *bias-free linear classifier* (CS231n's "each row of W is a classifier; the boundary is a hyperplane"), which is valuable — but it is **not** the manifold-untangling story the `learn`/`ml` text sells. → https://colah.github.io/posts/2014-03-NN-Manifolds-Topology/ and https://cs231n.github.io/linear-classify/

2. **The formula `new_representation = W·x (+ bias, + nonlinearity)` is shown, but the lab silently drops both the bias and the nonlinearity** — the two ingredients that make a "layer" more than a linear classifier. A learner finishing this boss could reasonably conclude a layer is just `W·x`. → A standard layer is `a = σ(W·x + b)`: Olah lists exactly three operations per layer — "(1) linear transformation by W, (2) translation by b, (3) point-wise application of tanh." The bias is *not* optional decoration: without it every decision boundary is forced through the origin (CS231n: "plugging in x=0 would always give score zero… all lines forced to cross the origin"). The nonlinearity is what gives the network its power. → https://colah.github.io/posts/2014-03-NN-Manifolds-Topology/ , https://cs231n.github.io/linear-classify/

3. **"a vertical line (x = 0) cleanly separates" + decision rule `x' > 0` is geometrically a linear classifier whose weight vector is the first row of W — this is worth naming, but the lesson never connects it.** The decision boundary in the *original* space is the line `a·x + b·y = 0`, i.e. row 1 of W is the classifier's normal vector. → CS231n: "every row of W is a classifier… the line shows all points that get a score of zero; the direction of increase shows the positive side." Making this explicit would turn a vague "tune four numbers" into "you are choosing a separating hyperplane's orientation." → https://cs231n.github.io/linear-classify/

4. **Mission 2 ("keep |det|>0.3 — no information loss") subtly overstates the link between det and information.** For *this 2D→2D* map, `det≠0` ⇔ invertible ⇔ no collapse, so the mission is fine here. But the `ml` note generalizes ("the determinant/eigen ideas tell you when those maps preserve or destroy information"), and **real layers are non-square** (e.g. 784→128). Square determinant is undefined for them; the right invariant is **rank** (and, quantitatively, singular values / the condition number from the SVD lesson). → A rectangular `W` can still lose information by being low-rank; det is a *special case*. Worth a one-line caveat so the master's-bound learner doesn't over-index on determinant. → https://cs231n.github.io/linear-classify/ (feature/representation framing) and general SVD/rank.

## Conceptual gaps (what a serious learner still needs)
- **The single most important missing idea: nonlinearity is mandatory.** Right now the boss can be "beaten" with pure linear algebra, which accidentally teaches that linear layers suffice. The learner should *experience the failure*: a dataset (concentric rings, or two interlocking moons, or XOR) that **no** setting of a linear W can separate, no matter the sliders.
- **Why composing linear maps gains nothing** (`W₂(W₁x) = (W₂W₁)x = Wx`). This is the crisp algebraic reason hidden layers need an activation between them.
- **What the bias does** (translates the boundary off the origin) — currently absent from the lab entirely.
- **The "untangling / manifold" picture** (Olah): a layer is a smooth deformation of space (a homeomorphism when invertible) that *bends and stretches* the input so classes pull apart; ReLU/tanh are the bends. The current text gestures at "edge space → shape space" but never shows space being warped.
- **Dimensionality can be a tool, not just a risk.** Olah's striking result: separating two concentric circles is impossible in 2D with an invertible map but trivial if you can lift to 3D (an extra hidden unit). The lesson treats added structure only through the det=0 *loss* lens and misses that *more* dimensions can *create* separability.
- **Square vs. rectangular layers / rank vs. determinant** (see issue 4) — needed before the learner meets real `784×128` weight matrices.

## Lab ideas (concrete: name the interactive, what the learner manipulates, what it reveals)

1. **"The Wall: linear can't win" (new mission on the existing lab, or a second dataset toggle).** Add a dataset selector: *Separable* (current clusters) and *Rings* (purple cluster inside an orange annulus, or two interlocking moons). On *Rings*, keep the same four W sliders. Mission text: "Try to separate these with W alone." It is **provably impossible** — after the learner flails, reveal a teaching card: "No linear map can do this. You need a nonlinearity." This is the missing emotional beat of the whole boss.

2. **"Add the bend" — a ReLU/tanh toggle.** Extend `T(p)` to `σ(W·x + b)` with a checkbox for nonlinearity and two bias sliders. On the *Rings* dataset, show that with one nonlinearity + a lifted coordinate the classes pull apart and the `x=0` cut suddenly works. Reveals: linear→stuck, +nonlinearity→solvable. Manipulates: 4 W sliders, 2 bias sliders, activation on/off, dataset.

3. **"See the warp" — render the transformed grid.** Already `plane.grid()` exists; draw the *image* of the gridlines under `T` (a deformed mesh), as Olah does. The learner literally watches space stretch/shear as they move sliders, and at `det≈0` watches the whole plane crush onto a line — making the collapse visceral rather than just a number going to 0.

4. **"Boundary in input space."** Draw the pre-image line `a·x + b·y = 0` in the *original* coordinates (the actual separating hyperplane), alongside the transformed view. Connects "I tuned a matrix" to "I oriented a classifier's decision boundary" (CS231n's row-of-W picture).

## Content improvements (specific learn/ml/deeper text upgrades)
- **Fix the central honesty gap.** Reframe the boss's first paragraph so the lab's actual scope matches the claim. Option A (recommended): change the narrative arc to *"first beat it with a linear map (you can, because this data is already separable) — then meet a dataset where linear ALONE fails, and add the nonlinearity."* That converts the inaccuracy into the lesson's climax.
- **Restore bias and nonlinearity to the body, not just the formula.** Add a sentence: "A real layer is `σ(W·x + b)`. The bias `b` slides the boundary off the origin; the nonlinearity `σ` (ReLU, tanh) is what lets stacked layers do more than one big matrix — without it, `W₂(W₁x)=(W₂W₁)x` collapses to a single linear map."
- **`ml` note:** add the composition-collapses line and soften the determinant claim — "for square maps, det=0 ⇔ collapse; for the rectangular layers in real nets, the right invariant is **rank** (low rank = lost directions), which the SVD lesson made precise." Keep the pixel→edge→shape chain but note each arrow is `σ(Wx+b)`, not bare `Wx`.
- **Add a `deeper` card on the untangling picture** (Olah): "A layer is a continuous deformation of space — it bends and stretches the input so tangled classes pull apart. When W is invertible the deformation is reversible (a homeomorphism); the nonlinearity supplies the bends a pure stretch can't." Cite the dimensionality insight: sometimes you must *add* a dimension to separate (concentric circles need ≥3 hidden units).

## Quiz improvements (specific misconceptions to target; keep questions self-contained)
- **Add the keystone question (currently the lesson's biggest blind spot):** "You stack two linear layers, `W₂(W₁x)`, with nothing in between. Compared to a single linear layer, this network can…" → correct: "compute nothing new — it equals one linear map `(W₂W₁)x`." Distractors: "separate twice as many classes," "always reduce error," "learn curved boundaries." WHY: composition of linear maps is linear; you need a nonlinearity between layers. (Self-contained — pure algebra.)
- **Add a nonlinearity question:** "Two classes form concentric rings (one inside the other). Can any single linear map `Wx` make them separable by a straight line?" → "No — no linear transform can untangle a class that surrounds another; a nonlinearity is required." Targets the exact misconception this lesson currently *reinforces*.
- **Add a bias question:** "Why include a bias `b` in `Wx + b`?" → "It lets the decision boundary not pass through the origin." Distractors: "it speeds training," "it normalizes inputs," "it prevents det=0." (Self-contained.)
- **Tighten the existing det=0 question** to flag it's the *square* case: keep it, but the WRONG_WHY for the "explode to infinity" distractor is already good. Consider adding "(for a square layer)" so it doesn't fight the rank caveat added to `ml`.
- All proposed questions are self-contained (no recall of lab-graph coordinates), per the curriculum's quiz rule.
