# c-rules — The Power Rule (Slopes for Free)

## Current summary (what it teaches + what the lab does)
The lesson introduces differentiation *rules* as a faster alternative to evaluating limits by hand. Core content:
- **Power rule**: `f(x)=xⁿ → f′(x)=n·xⁿ⁻¹`, presented as "exponent hops down front, drops by one" with worked cases (`x³→3x²`, `x²→2x`, `x→1`, constants `→0`).
- **Sum rule** `(f+g)′ = f′+g′` and **constant-multiple rule** `(c·f)′ = c·f′`, then a polynomial worked example `(x³+4x²−7x+2)′ = 3x²+8x−7`.
- **ml note**: autograd engines (PyTorch/JAX) store derivative rules per op; backprop replays them. ReLU's derivative (1 or 0) is cited as a reason it dominates DL.
- **Quiz** (3 Qs): `d/dx x⁵`; `d/dx 3x⁴−2x+7`; why a constant's derivative is 0. Each has tailored WRONG_WHY entries.
- **Lab `rules`** (`INTERACTIVES.rules`): plots `f(x)=xⁿ` (purple) and `f′(x)=n·xⁿ⁻¹` (cyan) on a shared plane. Chips select `n∈{1,2,3,4}`. The learner drags a gold tangent point along the curve; a gold tangent line and a readout (`f(x)`, `f′(x)`, x, slope) update live. Missions: (1) set n so slope at x=1 equals 3; (2) with n=2 find where slope=4; (3) pick n that makes f′ constant in x (n=1). No `deeper` cards exist for this lesson.

## Strengths
- The lab's central conceit — overlaying f′ as its own *curve* and showing "cyan height = gold line's slope" — is exactly the right way to make "the derivative is a function" concrete, and matches 3Blue1Brown's framing.
- Mission 3 (find the n whose slope doesn't depend on x) cleverly forces the learner to reason about f′ as a function of x rather than a number.
- WRONG_WHY entries are genuinely diagnostic: they name the specific slip (e.g. "brought the 5 down but kept the power at 5") rather than just restating the answer.
- The ml note correctly connects the rule table to autograd's op-level VJP registry and to ReLU's cheap subgradient — a real, load-bearing ML link.

## Inaccuracies / fidelity issues (each: the issue -> the correct statement -> source URL)
1. **"slopes follow patterns / rules turn differentiation into pattern-matching" with no proof or scope.** The power rule is presented purely as a mnemonic. -> A master's-bound learner should see *why* it's true and *for which exponents*. The clean derivation: for natural n, `(x+h)ⁿ = xⁿ + n·xⁿ⁻¹h + (terms in h²…)`; subtract `xⁿ`, divide by h, take h→0, and only the `n·xⁿ⁻¹` term survives. The rule then extends to all real exponents via `xʳ = e^{r ln x}` and the chain rule (valid for x>0). -> https://en.wikipedia.org/wiki/Power_rule , https://www.3blue1brown.com/lessons/derivatives-power-rule/
2. **Domain of the power rule is never stated.** The lesson implies `xⁿ → n·xⁿ⁻¹` always. -> The exponent's type matters: integers (incl. negative, via reciprocal rule) and rationals work on the natural domain, but for irrational/real exponents the identity `xʳ=e^{r ln x}` requires `x>0` (negative bases are not well-defined for irrational powers). Also the derivative at x=0 only exists for certain exponents. -> https://en.wikipedia.org/wiki/Power_rule
3. **`x → 1` is asserted, constants `→ 0`, but `x⁰` is glossed.** The quiz/WRONG_WHY note "7 = 7x⁰, so 7·0·x⁻¹ = 0," which is fine for x≠0 but `0⁰` is undefined — worth a footnote rather than presenting `7x⁰` as universally rigorous. -> `0⁰` has no consistent limit and is left undefined; the geometric argument (horizontal line ⇒ zero slope) is the clean justification, exactly as the lesson's quiz already says. -> https://en.wikipedia.org/wiki/Power_rule
4. **ReLU "derivative 1 (if x>0) or 0" stated as if total.** -> ReLU is *non-differentiable at 0*; frameworks use a **subgradient** (conventionally 0, sometimes 1) there. Calling it "the derivative" without noting the kink misses a point this curriculum elsewhere (ml-mlp) teaches visually. -> standard result; see e.g. https://en.wikipedia.org/wiki/Rectifier_(neural_networks)
5. **Lab readout claims the cyan curve "always equals the gold line's slope" — true, but for even n the displayed x range includes x<0 where the *visual* can mislead.** For n=2 the parabola is symmetric and f′=2x is an odd line through origin; learners may not connect negative slope on the left. Not an error in the math, but the note over-promises clarity without pointing at the sign of f′. -> tighten the note (see Lab ideas).

## Conceptual gaps (what a serious learner still needs)
- **No derivation at all.** The single most valuable upgrade: show the `(x+h)ⁿ` binomial collapse (or the geometric square/cube "nudge") so the rule is *discovered*, not memorized. 3B1B's square-area argument (`d(x²)=2x·dx+(dx)²`, drop `(dx)²`) and cube argument (`3x²·dx`) are the canonical intuitions and tie directly to "why the exponent becomes the coefficient." -> https://www.3blue1brown.com/lessons/derivatives-power-rule/
- **Why higher-order terms vanish.** The deep idea behind *all* of differential calculus — `(dx)²` is negligible vs `dx` — never appears. This is the same idea that later justifies linearization, Taylor's theorem, and Gauss–Newton.
- **Negative and fractional exponents.** `x⁻¹ → −x⁻²` and `√x = x^{1/2} → ½x^{-1/2}` are extremely common in ML (e.g. `1/√d` attention scaling, `x^{-1/2}` in normalization, `L2` norms). The lab caps n at integer {1,2,3,4} and never touches them.
- **Sum/constant-multiple rules are *linearity of the derivative operator*.** Framing d/dx as a linear operator (it commutes with + and scalar ·) is the bridge to viewing backprop/Jacobians as linear maps — worth one sentence.
- **Connection to gradients/Jacobians.** The lesson is scalar-only. A pointer that "the same rule applied coordinate-wise builds the gradient of `‖w‖²=Σwᵢ²` → `2w`, the workhorse of weight decay / ridge" would land the ML payoff hard.
- **Polynomial → general function caveat.** Learners over-generalize the power rule to `aˣ`, `xˣ`, products, and quotients. The lesson should explicitly fence the rule ("only for x raised to a constant power") to preempt the documented over-generalization error.

## Lab ideas (concrete: name the interactive, what the learner manipulates, what it reveals)
1. **`rules-derive` (Nudge the side) — new first lab.** A unit square of side x with a slider for x and a separate `dx` slider (large→small). Render the two `x·dx` rectangles and the `(dx)²` corner in distinct colors, with a live decomposition `Δarea = 2x·dx + (dx)²`. As `dx→0` the corner shrinks to nothing and the readout `Δarea/dx → 2x` converges to the analytic slope. Mission: "shrink dx until Δarea/dx is within 0.05 of 2x." Reveals the binomial/geometric origin of the coefficient. (Optionally a 3D-ish cube view for x³ → 3x².) Source: 3B1B square/cube argument.
2. **Upgrade `rules` to real exponents.** Replace the 4 integer chips with a continuous `n` slider spanning e.g. −2…3 (and snap markers at ½, 1, 2). Restrict plotting to x>0 when n is non-integer (and surface a small "defined for x>0" badge) — this *teaches the domain caveat by construction*. New mission: "set n so the slope at x=4 equals ¼" (forces `n=½`, `f′=½x^{-1/2}`, `f′(4)=¼`). Reveals fractional/negative-exponent behavior the current lab can't show.
3. **`rules-linearity` micro-lab.** Two draggable term-builders (pick coefficient + power for two monomials); show f, then f′ built term-by-term, with the operator note "d/dx is linear: it distributes over + and pulls out constants." Mission: build any f whose derivative at x=1 is exactly 0 (forces reasoning about cancellation, e.g. `x − x²` → `1 − 2x`, =−1 at 1; learner must hunt). Reinforces sum/constant-multiple as linearity.
4. **`rules-gradient-bridge` (optional, stretch).** `f(w)=‖w‖² = w₁²+w₂²` on a 2D plane; drag w, show gradient arrow `2w` and the per-coordinate power-rule computation. Connects the scalar power rule to the gradient used in weight decay. Mission: "drag w so the gradient has magnitude 2" (unit circle).

## Content improvements (specific learn/ml/deeper text upgrades)
- **Add a derivation paragraph** to `learn` before the rule table: "Where does it come from? Nudge x by a tiny `dx`. A square of side x grows by two strips `2x·dx` plus a negligible corner `(dx)²`; divide by dx, let dx→0, and `(x²)′=2x`. The same nudge on `xⁿ` (binomial expansion) leaves exactly `n·xⁿ⁻¹` — the exponent becomes the coefficient because there are *n* faces/strips."
- **State the scope** explicitly: "Works for any constant power — integer, negative (`x⁻¹→−x⁻²`), or fractional (`√x = x^{½} → ½x^{-½}`). For non-integer powers we need x>0. It does **not** apply to `aˣ` or `xˣ` (the base/exponent roles differ)."
- **Name linearity**: one line — "Sum and constant-multiple rules are just the statement that d/dx is a **linear operator**: it distributes over addition and lets scalars pass through. Backprop leans on this everywhere."
- **Sharpen the ml note on ReLU**: "ReLU is non-differentiable at 0; autograd uses a **subgradient** there (PyTorch returns 0). Away from 0 the slope is exactly 1 or 0 — cheap and non-vanishing on the active side, a big reason it beat sigmoid/tanh."
- **Add an ML-relevant example**: "`d/dw ‖w‖² = d/dw Σwᵢ² = 2w` — the gradient behind L2 regularization / weight decay is just the power rule applied per coordinate."
- **Add `deeper` cards** (currently none): (a) the binomial proof sketch; (b) extension to real exponents via `e^{r ln x}`; (c) the `(dx)²`-is-negligible principle and its reuse in Taylor/linearization.

## Quiz improvements (specific misconceptions to target; keep questions self-contained — never require recalling lab-graph data)
- **Keep** the three current questions — they're self-contained and target real slips (forgetting to drop the exponent, constant-multiple, constant→0). The WRONG_WHY coverage is strong.
- **Add: fractional exponent.** "`d/dx √x = ?`" options `½x^{-1/2}` / `2√x` / `½x^{1/2}` / `1/(2√x)·x` with answer `½x^{-1/2}` (note `=1/(2√x)`). Targets the documented "add instead of subtract the index" error and tests rewriting roots as powers.
- **Add: negative exponent.** "`d/dx 1/x = d/dx x⁻¹ = ?`" → `−x⁻²` (vs `x⁻²`, `−2x⁻³`, `ln x`). Targets sign error and shows the rule covers reciprocals.
- **Add: over-generalization trap.** "Which is NOT a valid use of the power rule?" with `d/dx x³`, `d/dx √x`, `d/dx 2ˣ`, `d/dx x⁻¹` → answer `d/dx 2ˣ` (constant base, variable exponent — the rule does not apply). Directly addresses the most-cited misconception (applying the power rule to exponentials).
- **Add: where does the coefficient come from?** A conceptual question — "In `x³→3x²`, the 3 appears because…" → "a small nudge to x grows the cube along 3 faces, each contributing x²·dx" (vs "it's a memorization trick", "3 is the next integer", "the chain rule"). Reinforces the new derivation, fully self-contained.
