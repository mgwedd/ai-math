# la-matcalc — Matrix Calculus: Shapes First

## Current summary (what it teaches + what the lab does)
The lesson generalizes scalar derivatives to vectors/matrices via one organizing idea: **derivatives have shapes**.
- Scalar L, vector x ∈ ℝⁿ ⇒ gradient ∇ₓL is a vector in ℝⁿ (same shape as x).
- Vector function f: ℝⁿ→ℝᵐ ⇒ Jacobian ∂f/∂x is m×n.
- Scalar L, matrix W (m×n) ⇒ ∂L/∂W is m×n (same shape as W), justified by the typecheck of W − η·∂L/∂W.
It then teaches the central deep-learning derivative for a linear layer y = Wx with upstream δ = ∂L/∂y:
∂L/∂W = δxᵀ (outer product, cell (i,j)=δᵢxⱼ) and ∂L/∂x = Wᵀδ. The `ml` note frames δxᵀ as a rank-1 update and ties it to LoRA. `deeper` cards give a "blame routing" reading and a note on numerator/denominator layout + the transpose chain.

The lab (`INTERACTIVES.lamatcalc`) has two modes:
- **Shapes mode**: sliders for n (inputs, 1–6) and m (outputs, 1–6) draw the forward pass y = Wx as colored cell grids plus the Jacobian ∂y/∂x and ∂L/∂W, both m×n. Mission: make the Jacobian tall (m > n).
- **Gradient δxᵀ mode**: sliders for x₁..x₃ and δ₁..δ₃ render the 3×3 outer-product grid, each cell colored/numbered by δᵢxⱼ, with a live rank readout. Missions: zero x₂ (its whole column darkens) and set all δ=0 (gradient vanishes → "converged").

## Strengths
- The "shapes first" framing is genuinely the right mental model and matches how CS231n teaches the affine backward pass ("the shape trick": dW must be X-shaped, so dW = Xᵀ·dout is the only contraction that typechecks).
- δxᵀ outer product taught visually with per-cell δᵢxⱼ is excellent; the "zero input ⇒ dark column" and "δ=0 ⇒ no gradient" missions build correct causal intuition (blame routing).
- Connecting the rank-1 structure of a per-example update to LoRA is a strong, accurate, master's-relevant hook.
- The numerator/denominator layout warning in `deeper` is the right caution and is rarely surfaced in intro material.

## Inaccuracies / fidelity issues (each: the issue -> the correct statement -> source URL)
- **Internal convention clash between the "gradient = same shape as x" rule and the "numerator layout" claim.** The lesson states ∇ₓL ∈ ℝⁿ (same shape as the column vector x) AND that the Jacobian ∂f/∂x is m×n in *numerator layout*. These two are different conventions. In strict numerator layout the gradient of a scalar is a **row** vector (1×n), i.e. the 1-row Jacobian; the "gradient is a column the same shape as x" is the **denominator / "gradient" convention.** The lesson silently mixes them. -> State explicitly: "We adopt the *gradient convention*: the derivative of a scalar w.r.t. any object has that object's shape (so ∂L/∂x is a column, ∂L/∂W is m×n). This is denominator layout. Many papers use numerator layout, where ∂L/∂x is a row and the Jacobian chain multiplies left-to-right instead." Parr & Howard use numerator layout and call the gradient a horizontal (row) vector. -> https://explained.ai/matrix-calculus/ and https://arxiv.org/pdf/1802.01528 (§"Generalization of the Jacobian"; "we are using the so-called numerator layout").
- **"∂y/∂x = W itself — the Jacobian" (shown in the lab) is convention-dependent and worth qualifying.** For y = Wx, the numerator-layout Jacobian ∂y/∂x equals W (m×n). That is correct in numerator layout — but the *same* lesson elsewhere uses the denominator/"same shape" convention for which ∂L/∂x = Wᵀδ. Learners need to see that ∂y/∂x = W (numerator) and ∂L/∂x = Wᵀδ are consistent precisely because the chain rule transposes: ∂L/∂x = (∂y/∂x)ᵀ δ = Wᵀδ. The lab asserts "∂y/∂x = W" without the transpose caveat that makes the two coexisting facts coherent. -> Add the one-line reconciliation. -> https://eli.thegreenplace.net/2018/backpropagation-through-a-fully-connected-layer/
- **The full Jacobian of ∂L/∂W is never characterized as a 4D object that we deliberately collapse.** The lesson jumps straight to δxᵀ. Authoritative treatments stress that ∂y/∂W is formally a rank-4 tensor (or, flattened, a huge sparse [m × mn] matrix) and that we *never materialize it*; δxᵀ is the contracted result. Omitting this hides *why* the outer product is the answer. -> "The exact Jacobian of a vector output w.r.t. a matrix is 4-dimensional and almost entirely zero (output i only touches row i of W). Contracting δ against it collapses to the outer product δxᵀ — we skip the tensor and write the result directly." -> https://eli.thegreenplace.net/2018/backpropagation-through-a-fully-connected-layer/ (the [T, NT] sparse-Jacobian / "160M elements, >0.5 GiB" discussion).

## Conceptual gaps (what a serious learner still needs)
- **Bias gradient.** Real layers are y = Wx + b. ∂L/∂b = δ (and ∂L/∂(Wx+b)/∂z passes through). The lesson never mentions b — a trivially small but conspicuous omission for a "the most important derivative in deep learning" lesson.
- **Batched matrix form.** The single-example δxᵀ should be lifted to the batch: for Y = XWᵀ (or Y = XW), ∂L/∂W = δ Xᵀ (sum of outer products), ∂L/∂b = Σ_b δ⁽ᵇ⁾ (column/row sum), ∂L/∂X = δ Wᵀ-style. The `ml` note *claims* "a batch sums B of them, rank ≤ B" but the actual matrix formula (dW = δ·Xᵀ as one matmul) is never written. CS231n's `dW = X.T.dot(dout)` is the canonical artifact to show.
- **Element-wise nonlinearity Jacobian is diagonal (Hadamard).** Backprop through an activation a = σ(z) is δ_z = δ_a ⊙ σ'(z), because the Jacobian of an element-wise map is diagonal. Parr & Howard make this a centerpiece. Without it, the learner cannot assemble a full layer (Wx+b → nonlinearity → next layer); the "chain of transposes" story is incomplete.
- **Why the gradient is the steepest-ascent direction / the inner-product (Riesz) reason the gradient lives in the same space as the variable.** The "shapes must match so the subtraction typechecks" argument is operational but not conceptual. The deeper reason ∂L/∂x shares x's shape is that the differential dL = ⟨∇L, dx⟩ is a linear functional represented by a vector in the same space.
- **Gradient checking (finite differences).** A master's-bound engineer should know to verify δxᵀ numerically: (L(W+εEᵢⱼ)−L(W−εEᵢⱼ))/2ε ≈ ∂L/∂Wᵢⱼ. This is how every framework's autograd is unit-tested.
- **Softmax/cross-entropy Jacobian** as the canonical *non-diagonal* vector→vector Jacobian (and the famous δ = ŷ − y simplification) would be a natural capstone connecting Jacobian shape to a real loss.

## Lab ideas (concrete: name the interactive, what the learner manipulates, what it reveals)
- **"Full layer backward" mode (extend the existing lab).** Add a third mode that shows the complete pipeline x → z = Wx + b → a = σ(z) → L. Learner manipulates δ_a (upstream) and toggles the activation (ReLU/tanh). The lab renders δ_z = δ_a ⊙ σ'(z) as an element-wise *diagonal* Jacobian (light up only the diagonal), then ∂L/∂W = δ_z xᵀ. Reveal: the diagonal-Jacobian fact and how a nonlinearity just *gates* the upstream gradient. Mission: "kill a unit with ReLU (z<0) and watch its entire row of ∂L/∂W go dark" — contrasts nicely with the existing "zero input ⇒ dark column."
- **"Batch = sum of rank-1 layers" animator.** In Gradient mode, add a B slider (1–4). Show B separate δ⁽ᵇ⁾xᵀ⁽ᵇ⁾ grids stacking/summing into the batch gradient, with a live `rank ≤ B` readout that actually computes the numeric rank of the sum. Reveal: the rank-≤B claim from the `ml` note becomes visible and verifiable; directly motivates LoRA's rank-r bet.
- **"Shape detective / convention flipper" toggle in Shapes mode.** A switch between numerator and denominator layout that transposes every drawn object and updates the chain-rule arrow direction. Reveal: the same math, two conventions, and *why reading a paper requires recovering the convention from one known shape* (the `deeper` card's point, made tactile).
- **Numeric gradient-check overlay.** In Gradient mode, add a "perturb Wᵢⱼ" affordance: clicking a cell shows the finite-difference estimate next to the analytic δᵢxⱼ, confirming they match. Reveal: gradients are checkable, not magic.

## Content improvements (specific learn/ml/deeper text upgrades)
- **State the convention up front.** Add one sentence: "Throughout, we use the *gradient convention* — the derivative of a scalar has the shape of the thing you differentiate by. (Some texts use numerator layout, where gradients are rows; then the chain rule multiplies the other way.)" This removes the silent clash flagged above.
- **Add the bias and the batch.** Extend the formula block: "Full layer: z = Wx + b ⇒ ∂L/∂b = δ. Batch of B examples (rows of X): ∂L/∂W = δXᵀ = Σ_b δ⁽ᵇ⁾x⁽ᵇ⁾ᵀ, ∂L/∂b = Σ_b δ⁽ᵇ⁾." Note this is literally `dW = X.T @ dout` in any framework.
- **Add the activation / diagonal-Jacobian line.** "Through an element-wise activation a = σ(z), the Jacobian is diagonal, so the gradient just gets gated: δ_z = δ_a ⊙ σ'(z). Stacking layers alternates Wᵀ (mixing) with ⊙σ' (gating) — that's the whole backward pass."
- **Reconcile ∂y/∂x = W with ∂L/∂x = Wᵀδ explicitly** in the `deeper` chain-rule card: "∂y/∂x = W (numerator layout); the chain rule transposes it: ∂L/∂x = (∂y/∂x)ᵀδ = Wᵀδ. Same fact, the transpose is the bridge."
- **`ml` note:** keep the LoRA framing but add the batch matrix form so the "rank ≤ B" claim is anchored to δXᵀ, not just prose. Optionally note that *gradient accumulation* over microbatches is exactly summing these rank-1 (or low-rank) updates.
- **Add a "why same space" sentence** to the blame card or a new card: the gradient lives in x's space because dL = ⟨∇L, dx⟩ — a Riesz-representation reason beyond "so subtraction typechecks."

## Quiz improvements (specific misconceptions to target; keep questions self-contained — never require recalling lab-graph data)
Existing four questions are good and self-contained. Suggested additions/edits:
- **Bias gradient.** "For z = Wx + b with δ = ∂L/∂z, the gradient ∂L/∂b is…" → answer δ (same shape as b); distractors: Wᵀδ, δxᵀ, a scalar Σδ. Targets the omission and the bias/weight confusion.
- **Element-wise activation.** "Backprop through a = σ(z) (element-wise) sends the upstream gradient δ_a back as…" → δ_a ⊙ σ'(z); distractors: σ'(z)·δ_a as a matrix product, J σ'(z) with full dense J. Targets the diagonal-Jacobian misconception.
- **Batched weight gradient.** "A batch X (B rows) gives ∂L/∂W = …" → δXᵀ (a sum of B outer products); distractors: the per-example δxᵀ unchanged, an average without the sum, Xᵀδ (wrong order). Targets the single-vs-batch and order confusion.
- **Convention probe (self-contained).** "A paper writes ∂y/∂x as n×m for y∈ℝᵐ, x∈ℝⁿ. Which layout is it using?" → denominator layout (gradient = same shape as x). Reinforces the `deeper` card without referencing the lab.
- **Edit WRONG_WHY for Q3 distractor 'Wᵀδ':** it already correctly says this is ∂L/∂x; consider adding "— and note ∂y/∂x = W, so Wᵀ appears only after the chain rule transposes it," to pre-empt the numerator/denominator confusion.
